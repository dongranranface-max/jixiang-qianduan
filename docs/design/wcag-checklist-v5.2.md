## 一、对比度要求（1.4.3 对比度）

### 1.1 文字对比度

| 检查项 | WCAG 标准 | 测量方法 | 常见问题 |
|--------|-----------|---------|---------|
| 正文文字（< 18px） | ≥ 4.5:1 | Lighthouse Contrast | $text-muted 在 $bg-primary 上 |
| 大文字（≥ 18px 或 ≥ 14px bold） | ≥ 3.0:1 | Lighthouse Contrast | - |
| UI 组件和图形对象 | ≥ 3.0:1 | 手动检查 | 按钮边框/输入框边框 |

### 1.2 ⚠️ 重点检查项（v5.2 修复）

| Token | 浅色背景组合 | 对比度 | 状态 |
|-------|-------------|--------|------|
| $gold (#C4A57B) on $bg-primary (#F8F7F4) | 浅色 | **3.2:1 ❌** | 已修复：使用 $gold-on-light (#9B7E5E) |
| $text-muted (#8A8A8A) on $bg-primary (#F8F7F4) | 浅色 | 4.8:1 ✅ | 合规 |
| $text-secondary (#5C5C5C) on $bg-primary (#F8F7F4) | 浅色 | 7.5:1 ✅ | 合规 |
| $text-primary (#141414) on $bg-secondary (#FFFFFF) | 深色 | 16.1:1 ✅ | 合规 |
| $success (#3D8B6E) on $bg-primary (#F8F7F4) | 浅色 | 5.9:1 ✅ | 合规 |
| $warning (#D4850A) on $bg-primary (#F8F7F4) | 浅色 | 4.8:1 ✅ | 合规 |
| $danger (#C0454A) on $bg-primary (#F8F7F4) | 浅色 | 4.6:1 ✅ | 合规 |

### 1.3 对比度修复代码对照

```scss
// ❌ 错误：浅色背景使用 $gold
.text-on-light {
  color: $gold;  // 对比度 3.2:1 ❌
}

// ✅ 正确：浅色背景使用 $gold-on-light
.text-on-light {
  color: $gold-on-light;  // 对比度 5.3:1 ✅
}

// ✅ 深色背景可以继续使用 $gold
.text-on-dark {
  color: $gold;  // 对比度合规 ✅
}
```

## 二、可操作性要求（2.1 可点击目标）

### 2.1 最小点击区域

| 组件 | WCAG 标准 | 集享公社实现 | 状态 |
|------|----------|-------------|------|
| 按钮高度 | ≥ 44×44px（建议 ≥ 48×48px） | 88rpx ≈ 44px ✅ | 合规 |
| TabBar 项 | ≥ 44×44px | 100rpx 含安全区 ✅ | 合规 |
| 列表项 | ≥ 44×44px | 80rpx+ ✅ | 合规 |
| 图标按钮 | ≥ 44×44px | 48rpx 图标 + padding ✅ | 合规 |

### 2.2 间距要求

```scss
// 列表项之间保持足够间距，防止误触
.list-item {
  min-height: 80rpx;  // 防止误触相邻项
  padding: $space-base 0;  // 上下间距
}
```

## 三、语义化要求（1.3.1 信息结构）

### 3.1 表单关联

| 要求 | 实现方式 | 检查点 |
|------|---------|--------|
| label 关联 | `for` / `id` 属性 | 每个输入框必须有对应 label |
| aria-label | 无可见 label 时使用 | 图标按钮/纯图标操作 |
| aria-describedby | 错误提示关联 | 输入框错误状态 |
| role | 语义化角色 | 按钮/链接/列表等 |

```html
<!-- ✅ 正确示例 -->
<label for="phone-input">手机号</label>
<input id="phone-input" type="tel" aria-describedby="phone-error" />
<span id="phone-error" class="error-msg">手机号格式错误</span>

<!-- ✅ 图标按钮 -->
<button aria-label="收藏" @click="toggleFavorite">
  <icon :name="isFavorited ? 'star-filled' : 'star'" />
</button>
```

### 3.2 图片替代文本

| 图片类型 | alt 属性要求 |
|---------|------------|
| 功能性图片（按钮/图标） | 必须有描述性 alt |
| 装饰性图片 | alt=""（空字符串） |
| 内容图片（商品图） | 简短描述 alt="商品名称图片" |
| Logo | alt="集享公社" |

### 3.3 动态内容通知

```html
<!-- 新内容加载完成 -->
<div aria-live="polite" aria-atomic="true">
  <!-- 内容更新时屏幕阅读器会通知用户 -->
</div>

<!-- 重要通知（错误/警告）-->
<div role="alert">
  <!-- 立即通知用户 -->
</div>
```

## 四、键盘操作要求（2.1 键盘可访问）

### 4.1 焦点可见性

| 要求 | 实现 | 状态 |
|------|------|------|
| Tab 焦点可见 | 金色 outline 边框 | ✅ |
| 焦点不遮挡内容 | outline-offset | ✅ |
| 焦点顺序 | 符合视觉顺序 | 需测试 |
| 焦点陷阱 | Modal 弹出后焦点锁定 | 需实现 |

```scss
// 焦点样式
*:focus-visible {
  outline: 2rpx solid $gold-on-light;
  outline-offset: 4rpx;
}

// 移除默认 focus 样式（移动端）
*:focus {
  outline: none;
}
```

### 4.2 键盘导航顺序

```
Tab 顺序规范：
1. 页面顶部 Skip Link（可选）
2. 顶部导航/搜索（如有）
3. 主要内容区的第一个可交互元素
4. ...
5. TabBar 导航项
6. 页脚（如有）

顺序原则：符合视觉从上到下/从左到右的阅读习惯
```

### 4.3 快捷键支持

| 操作 | 快捷键 | 备注 |
|------|--------|------|
| 搜索框聚焦 | / | 页面任意位置按 / 跳到搜索 |
| 关闭弹框 | ESC | Modal/Toast 等 |
| 确认操作 | Enter | 按钮聚焦时 |
| 返回 | Backspace | 部分场景 |

## 五、页面结构（1.3.2 意义顺序）

### 5.1 标题层级

| 元素 | Tag | 要求 |
|------|-----|------|
| 页面标题 | h1 | 每页必须有且仅有一个 h1 |
| 区块标题 | h2 | 同级区块标题并列 |
| 子区块标题 | h3 | h2 的子级 |
| 禁止 | h4-h6 | 保持 h1-h3 三层结构 |

### 5.2 语义化区域

```html
<header>    <!-- 顶部导航/资产卡片 -->
<main>      <!-- 主要内容区 -->
<nav>       <!-- TabBar 导航 -->
<aside>     <!-- 侧边栏（如有筛选面板）-->
<footer>    <!-- 页脚 -->
```

## 六、退款/超时/不可用状态

### 6.1 服务不可用

| 场景 | 要求 | 实现 |
|------|------|------|
| 网络断开 | 显示状态 + 替代内容 | 网络异常横幅 + 缓存数据展示 |
| 加载失败 | 显示错误 + 重试按钮 | 错误插画 + 「重试」按钮 |
| 服务维护 | 显示维护公告 | 维护状态页 |
| 功能不可用 | 显示原因 + 引导 | disabled 态 + tooltip 说明 |

### 6.2 时间限制

| 要求 | 实现 |
|------|------|
| 倒计时可见 | 屏幕阅读器可读（aria-label） |
| 倒计时结束 | 自动处理或通知用户 |
| 时间延长选项 | 提供延长入口（如有） |

## 七、测试方法

### 7.1 自动化测试

```bash
# Lighthouse 无障碍审计
# Chrome DevTools → Lighthouse → Accessibility

# axe-core 浏览器插件
# 安装后右键 → Inspect → 查看无障碍问题

# 关键指标
- Accessibility Score ≥ 90
- 无 Critical/Serious 问题
```

### 7.2 手动测试清单

```
□ 纯键盘操作：只用 Tab/Enter/Esc 能完成核心流程
□ 屏幕阅读器测试：NVDA/VoiceOver 能否正确朗读
□ 放大 200% 测试：文字是否截断/重叠
□ 高对比度模式：Windows High Contrast / Safari 强对比
□ 动态内容：aria-live 是否正常工作
□ 表单错误：错误提示是否清晰/关联是否正确
□ 焦点管理：Modal 弹出后焦点是否锁定
□ 跳过链接：是否有「跳转到主要内容」链接
```

### 7.3 真机测试

| 平台 | 测试工具 |
|------|---------|
| iOS | VoiceOver（设置 → 辅助功能 →  VoiceOver）|
| Android | TalkBack（设置 → 辅助功能 → TalkBack）|
| Windows | NVDA 屏幕阅读器 |
